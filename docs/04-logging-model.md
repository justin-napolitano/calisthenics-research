# Logging Model

The log should capture enough data to explain future changes without turning the routine into paperwork.

## Required Fields

- Date.
- Session type: `hard`, `easy`, `recovery`, `walk_only`, or `missed`.
- Baseline completed: yes/no.
- Exercises, reps, rounds, and plank times.
- Effort rating from 1 to 10.
- Soreness rating from 0 to 10.
- Pain signals: none or specific location.
- Short note.

## Optional Fields

- Sleep quality.
- Bodyweight.
- Steps or walk duration.
- Mood or stress.
- Rest times.
- Variation used for each exercise.

## Review Questions

At weekly review, answer:

- How many days were logged?
- How many hard days were completed?
- Which movement was easiest?
- Which movement was most limiting?
- Did any joint or pain signal repeat?
- Did easy days help recovery or become hidden hard days?
- What is the smallest useful adjustment?
